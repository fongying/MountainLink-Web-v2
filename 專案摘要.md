# Project Summary (for AI Handoff)

Project: MountainLink

Purpose:
- Blue Force Tracker style monitoring platform for mountain/rescue operations.
- Track field devices in real time, show status on map, and support emergency coordination.

Architecture:
- Device side: ESP32 + LoRa (Meshtastic ecosystem, external to this web repo)
- Messaging: MQTT (mosquitto, external integration path)
- Backend options in platform context: Fastify + Postgres (external/legacy path)
- Current web app stack (this repo): SvelteKit + TypeScript + SQLite
- Ingestion option: Node-RED (external integration path)
- Map: Google Maps JS API (2D + 3D)
- Real-time channel: Server-Sent Events (SSE)
- Weather/EQ data source: CWA Open Data API

Key Topics:
- device telemetry
- real-time map
- rescue system
- unit classification (hiker / victim / SSRT / police-fire / volunteer)
- account-device binding
- natural hazard alerts (rain / low temperature / earthquake)

Current task:
- Integrate earthquake notifications into the unified "Natural Hazard Alert Zone".
- Keep earthquake feed concise: only events from last 3 days, max 3 items.
- Maintain a single alert surface for rain, low-temperature, and earthquake.

Problem:
- Some pages/components still have mojibake (text encoding corruption) and should be normalized to UTF-8.
- Earthquake trigger pipeline currently stores state in memory (non-persistent after server restart).
- Earthquake SSE push events are not yet integrated into `/api/stream` (currently polling in UI).
- Webhook security for trigger intake requires `MLINK_WEBHOOK_SECRET` to be configured.

Key APIs (current):
- `GET /api/alerts/rain`
- `GET /api/alerts/cold`
- `GET /api/eq/events`
- `GET /api/eq/latest`
- `POST /api/hooks/eq/trigger` (HMAC verified)

Important paths:

Data storage in this repo:
